Manual de uso de TecXPaneL

Dashboard

Muestra el estado del servidor en tiempo real: uso de CPU, RAM, disco y red. También ves los servicios activos (Nginx, MySQL, etc.) y los procesos que más recursos consumen. Los datos se actualizan automáticamente cada pocos segundos.

Terminal

Una terminal SSH real dentro del navegador. Puedes ejecutar cualquier comando como si estuvieras conectado por SSH. Pulsa Conectar para abrir la sesión. Soporta colores, autocompletado y todo lo que harías en una terminal normal.

Logs del sistema

Consulta en vivo los registros de Nginx y del sistema, y el registro de auditoría del panel: cada acción relevante (inicios de sesión, creación o borrado de sitios, apps, bases de datos, cambios de firewall…) queda anotada con usuario, IP y fecha. Útil para diagnosticar errores y revisar quién hizo qué.

Sitios web

Crea y gestiona sitios web alojados en tu VPS. Puedes crear sitios de tipo:

  • HTML estático — páginas simples sin backend.
  • PHP — WordPress, Laravel y cualquier app PHP. Puedes elegir la versión de PHP.
  • Node.js / React / Python — se configura como proxy inverso hacia tu app.

Sin dominio: Si no tienes un dominio, marca «Acceso por IP:Puerto» y el panel asignará un puerto automáticamente. Podrás acceder al sitio desde http://tu-ip:puerto.

Con dominio: Introduce tu dominio (ej: ejemplo.com). Recuerda crear un registro A apuntando a la IP de tu VPS. Luego puedes instalar SSL desde la sección SSL/HTTPS.

Aplicaciones

Despliega aplicaciones Node.js, Python, TypeScript o React gestionadas por PM2. El panel se encarga de mantenerlas vivas y reiniciarlas si se caen.

  • Ruta del proyecto: la carpeta donde está tu código (ej: /var/www/mi-api).
  • Comando de inicio: puede ser un archivo (server.js, app.py) o un comando npm (npm start, npm run dev). Es editable: si la app no levanta, corrige el comando y reinicia.
  • Puerto: en qué puerto escucha tu app (informativo, para configurar el proxy).

Origen del código: sube un .zip/.tar.gz o clona desde Git. Si usas Git, el panel genera un webhook de auto-deploy: cada git push dispara automáticamente pull → instalar → build → reinicio.

Python: cada app se aísla en su propio virtualenv (.venv). El panel distingue un servicio web (Flask/FastAPI con puerto y proxy Nginx) de un worker/bot sin puerto (p. ej. un bot de Telegram), donde eliges el script .py de arranque.

Variables de entorno: gestiona el archivo .env de la app desde el propio panel.

Bases de datos

Crea bases de datos MySQL (MariaDB) o PostgreSQL con un clic. El panel genera automáticamente un usuario y contraseña seguros. Las contraseñas se almacenan cifradas y puedes copiarlas con el botón de copiar.

Requisito: el motor de BD debe estar instalado. Si no lo instalaste durante el setup, puedes hacerlo desde Plugins.

phpMyAdmin: para gestionar tus bases de datos visualmente, instala phpMyAdmin desde Plugins. Estará accesible en el puerto 8081.

Docker

Gestiona contenedores Docker sin usar la terminal: lístalos con su estado y puertos, arráncalos, deténlos, reinícialos, elimínalos y consulta sus logs en vivo.

  • Crear un contenedor desde una imagen del registro o compilando un Dockerfile, con mapeo de puertos, variables de entorno, volúmenes persistentes y proxy Nginx + SSL opcional por dominio.
  • Editor de Dockerfile y docker-compose con despliegue en un clic (docker compose up -d).

Requisito: Docker instalado (un clic desde Plugins). Es la misma base sobre la que corren Workflows y Correo.

Workflows (n8n)

Automatiza flujos con n8n sin instalarlo a mano. Pulsa Instalar n8n y el panel lo levanta como contenedor Docker (con volumen persistente y proxy Nginx opcional), mostrando el progreso de descarga.

  • Abre n8n, crea tu cuenta y genera tu API key en Settings → API; pégala en el panel (se guarda cifrada).
  • Desde el panel: lista tus workflows, actívalos/desactívalos, revisa las ejecuciones recientes y abre el editor de n8n con un clic.

Requisito: Docker instalado.

Copias de seguridad

Crea backups completos o por recurso (bases de datos, sitios, apps, config del panel). Cada backup se guarda como .tar.gz con un manifest.json y puedes descargarlo.

  • Restauración granular: restaura todo el backup o una pieza suelta. Antes de sobrescribir, el panel crea un snapshot de seguridad de lo que va a reemplazar.
  • Programación: backups automáticos por cron (diario/semanal, hora y retención).
  • Destinos remotos (opcional): replica los backups fuera del VPS en S3-compatible (Backblaze B2, Wasabi, MinIO, Spaces…) o SFTP vía rclone, con cifrado opcional mediante tu passphrase.

Aviso: los backups locales incluyen el .env y los dumps de tus bases de datos sin cifrar. Guárdalos por canales seguros o usa un destino remoto con cifrado. Si activas el cifrado, guarda la passphrase fuera del VPS.

Tareas programadas (Cron)

Crea y gestiona tareas cron desde la UI con un constructor guiado (cada hora, cada día a las HH:MM, cada semana, cada mes, o modo avanzado con los cinco campos cron).

  • Activa/desactiva una tarea sin borrarla, edítala y consulta el log de salida de cada una.
  • El panel gestiona solo sus propias tareas: las líneas que pusiste a mano o las de backups se respetan.

Correo

Servidor de correo autohospedado con docker-mailserver (Postfix + Dovecot + Rspamd + DKIM) en un contenedor, gestionado desde el panel.

  • Pulsa Instalar, indica el hostname (ej. mail.tudominio.com) y el panel emite el certificado TLS con Certbot.
  • Crea buzones y alias, genera el DKIM y consulta los registros DNS a añadir: MX, SPF, DKIM, DMARC (y el PTR/rDNS en tu proveedor de VPS).

Aviso: el correo no funciona hasta que el DNS sea correcto (sobre todo MX y PTR). Requiere Docker y ~1 GB de RAM para el contenedor.

DNS autoritativo

Convierte tu VPS en un servidor DNS autoritativo con PowerDNS para gestionar el DNS de tus dominios desde el panel.

  • Pulsa Instalar PowerDNS (abre el puerto 53) y configura tus nameservers (ns1/ns2) apuntando a la IP del VPS.
  • Añade tus dominios (zonas) y gestiona sus registros (A, AAAA, CNAME, MX, TXT).
  • El panel te muestra los glue records a crear en tu registrador y verifica si la delegación ya está activa.

Aviso: crear los glue records y cambiar los nameservers en el registrador del dominio lo haces tú (es externo al VPS).

Gestor de archivos

Navega, crea, edita y elimina archivos y carpetas del sistema. Ideal para subir o editar el código de tus sitios web directamente desde el navegador.

Firewall UFW

Gestiona las reglas del firewall UFW. Por defecto se abren los puertos 22 (SSH), 80 (HTTP) y 443 (HTTPS). Puedes añadir reglas para abrir otros puertos que necesiten tus apps.

SSL / HTTPS

Instala certificados SSL gratuitos de Let’s Encrypt en tus sitios con dominio. El certificado se renueva automáticamente. Requisitos: tener un dominio apuntando a la IP del VPS y Certbot instalado (viene por defecto o se instala desde Plugins).

Plugins

Instala software adicional sin tocar la terminal:

  • Docker — contenedores para aislar aplicaciones.
  • phpMyAdmin — gestor web para MySQL/MariaDB (puerto 8081).
  • Redis — caché en memoria para mejorar rendimiento.
  • Fail2Ban — protege SSH y otros servicios contra ataques de fuerza bruta.
  • Composer — gestor de paquetes PHP (Laravel, WordPress plugins).
  • Certbot — certificados SSL gratuitos.

Ajustes

Cambia tu contraseña de acceso al panel (mínimo 8 caracteres) y protege tu cuenta:

  • 2FA (TOTP): activa un segundo factor compatible con Google Authenticator, Authy y similares escaneando un código QR. Al iniciar sesión se te pedirá el código de 6 dígitos.
  • Recuperación de contraseña: configura un email y una pregunta de seguridad para poder recuperar el acceso si la olvidas, sin depender de servicios externos.
  • Notificaciones: conecta un bot de Telegram (creado con @BotFather) y/o un SMTP para recibir avisos de disco lleno, servicios caídos y eventos de seguridad, con mensaje de recuperación. Si el VPS entero se cae, usa un monitor externo.
